分页查询的方式：

	- PageResult类 是用于获取后端查询结果，返回到Controller，再传到前端的类
	- 分页查询DTO类 是用于接收前端传来的查询条件，页码，单页数据量的参数的，然后这些参数会作为之后的PageHelper的传参以及Mapper类的传参被使用

	1.首先创建PageResult类用来封装查询结果，变量一般为 long total; list records;
							    					总记录数       数据集合

	2.创建分页查询的DTO类，以做到接收前端请求，精准封装，变量一般为 查询条件(id等)，int page; int pageSize(单页数据量)
												↓
											前端将请求传入DTO类
	3.Controller层
		    public Result<PageResult> page(EmployeePageQueryDTO employeePageQueryDTO) {
				
        		log.info("员工分页查询: {}", employeePageQueryDTO);

        		return Result.success(employeeService.pageQuery(employeePageQueryDTO));
    		}
	
	4.Service层
			public PageResult pageQuery(EmployeePageQueryDTO employeePageQueryDTO){

				//  1.设置分页查询参数：page，pageSize
				PageHelper.startPage(employeePageQueryDTO.getPage(), employeePageQueryDTO.getPageSize());

				//  2.调用Mapper方法，将查询结果代入PageHelper提供的Page类里  Page类本质是一个list集合
				Page<Employee> p = employeeMapper.pageQuery(employeePageQueryDTO);

				//  3.封装并返回结果
				return new PageResult(p.getTotal(), p.getResult());
			};

	5.Mapper层
			Page<Employee> pageQuery(EmployeePageQueryDTO employeePageQueryDTO);
