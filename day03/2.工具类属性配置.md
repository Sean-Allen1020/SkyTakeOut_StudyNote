工具类，顾名思义，提供特定功能的类。使用者通过简单的方法调用，达成所需的处理。
为了保证工具类的使用便利性，一般会选择`static`来修饰方法，省去调用者new实例的步骤。

然而，有的工具类需要对一些属性的值进行预设，而属性又必须设置为private的情况下

- 当某个“工具/组件”需要依赖外部配置（如 yml 中的属性配置）时，不建议做成纯`static工具类`；更推荐交给`IOC容器`创建并注入。
  - 可以通过`@Configuration + @Bean`的方式交给IOC容器管理，在配置层集中完成实例化与参数装配。

- 配置类的具体写法：
  - 示例工具类： 
```java
    @Data
    @AllArgsConstructor
    public class AliOssUtil {

        private String endpoint;
        private String bucketName;
        private String region;
        private String accessKeyId;
        private String accessKeySecret;

        public String upload(MultipartFile file) throws com.aliyuncs.exceptions.ClientException {
            // ...
            return "...";
        }
    }
```
  - 配置类：
```java
    @Configuration
    public class AliOssConfiguration {

        @Bean                                       //这里调用的properties是实现准备好的配置了属性值的封装类，配置方式是通过yml，具体在"0.基于yml配置实体类.md"中查看
        public AliOssUtil aliOssUtil(AliOssProperties aliOssProperties) {

            return new AliOssUtil(aliOssProperties.getEndpoint(),
                            aliOssProperties.getBucketName(),
                            aliOssProperties.getRegion(),
                            aliOssProperties.getAccessKeyId(),
                            aliOssProperties.getAccessKeySecret());
        }
    }
```
