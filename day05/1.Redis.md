### Redis简介
- 是一种基于内存的key-value结构的数据库
  - 基于内存存储，读写性能更高
  - 由于内存本身有限，所以适用场景也有限--多用于存储**热点数据**(热点商品,咨询,新闻)之类，在**短时间内**有大量访问的数据

- Redis存储的是key-value结构的数据，其中key是字符串类型，value有5种常见类型
  - 字符串`String`  ：最简单的值。
  - 哈希`hash`      ：内部又包括一组`String类型`的键值对，形式类似于`HashMap`。通常用于存储`*对象*
  - 列表`list`      ：按照插入顺序排序，可从左插入也可从右插入，类似`LinkedList`，首尾操作，增删快。
  - 集合`set`       ：`String类型`的无序集合，没用重复元素，类似`HashSet`。可用于*交集运算* -- Java中的交集运算为`set1.retainAll(set2)`
  - 有序集合`sorted set/zset`：集合中每个元素关联一个分数(score)，根据分数升序排序，没有重复元素。可用于*排行榜功能*的开发

-----------------------------------------------------------------------------------------
### Redis 数据结构是否允许重复对照表

| 数据类型          | 是否允许重复                          | 说明                              |
| ----------------- | ------------------------------------- | --------------------------------- |
| String            | ❌ Key 不允许重复<br>✅ Value 可重复    | 相同 key 会直接覆盖旧值           |
| Hash              | ❌ Field 不允许重复<br>✅ Value 可重复  | 相同 field 会覆盖 value           |
| List              | ✅ 允许重复                            | 有序，可存相同元素多次            |
| Set               | ❌ 不允许重复                          | 天生去重，常用于好友、标签        |
| Sorted Set (ZSet) | ❌ Member 不允许重复<br>✅ Score 可重复 | 相同 member 会保留 score 高的一个 |

### Redis常用命令
**指令中大写部分为语法结构，小写部分为自定义内容**

#### 字符串操作命令
*key和value的位置用于写上自己定义的键值对*

- SET key value             设置指定key的值
- GET key                   获取指定key的值
- SETEX key seconds value   设置指定key的值，并将key的过期时间设为seconds秒(验证码)
- SETNX key value           只有在key不存在时设置key的值

-----------------------------------------------------------------------------------------
#### 哈希操作命令
*key是Redis原本的键，field和value是对象的属性和值*

- HSET key field value      将哈希表key中的字段field的值设为value
- HGET key field            获取存储在哈希表中指定字段的值
- HDEL key field            删除存储在哈希表中的指定字段
- HKEYS key                 获取哈希表中的所有字段
- HVALS key                 获取哈希表中的所有值

-----------------------------------------------------------------------------------------
#### 列表操作命令
*L代表left，也就是以表首为基准操作。**表尾的情况则用R**，即right*

- LPUSH key value1 value2 ...   将一个或多个值插入到列表头部(类似栈)
- LRANGE key start stop         获取列表指定范围内的元素
  - LRANGE key 0 -1 (表示获取从头到尾的元素，-1表示尾)
- LPOP key                      移除并获取列表头部元素
- LLEN key                      获取列表长度

-----------------------------------------------------------------------------------------
#### 集合操作命令
- SADD key member1 member2 ...  向集合添加一个或多个成员
- SMEMBERS key                  获取集合中的所有成员
- SCARD key                     获取集合的成员数量
- SINTER key1 key2 ...          获取所有指定集合的交集
- SUNION key1 key2 ...          获取所有指定集合的并集
- SREM key member1 member2 ...  删除集合中一个或多个成员

-----------------------------------------------------------------------------------------
#### 有序集合操作命令
*相同member会保留score高的一个，相同score不同member可共存*

- ZADD key score1 member1 score2 member2 ...  向有序集合添加一个或多个成员(及分数)  
- ZRANGE key start stop WITHSCORES            通过索引区间返回有序集合中的成员
- ZINCRBY key increment member                有序集合中对指定成员的分数加上增量
- ZREM key member1 member2 ...                移除有序集合中的一个或多个成员

-----------------------------------------------------------------------------------------
#### 通用命令
- KEYS pattern    查找所有符合指定模式的key
  - KEYS *    查找所有key
  - KEYS abc* 查找所有abc开头的key
- EXISTS key      检查指定key是否存在
- TYPE key        返回key所储存的值的类型
- DEL key         在指定key存在时删除key
